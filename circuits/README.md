# ZK-DocAuth Circom Circuits

Zero-knowledge circuits for the ZK-DocAuth credential verification protocol.

## Circuit: `zk_doc_auth.circom`

**Constraint count:** ~25,000 (dominated by 20-level Poseidon Merkle proof)

### What the circuit proves (without revealing private data):

1. **Credential Commitment** — The prover knows credential fields whose Poseidon hash matches a Merkle leaf.
2. **Merkle Inclusion** — That leaf exists in a tree whose root is publicly known.
3. **Age ≥ min_age** — Date-of-birth satisfies an age threshold (range proof via bit decomposition).
4. **Nationality Match** — `Poseidon(nationality) == expected_nationality_hash`.
5. **Document Not Expired** — `expiry > current_date`.

### Signals

| Signal | Visibility | Description |
|---|---|---|
| `name` | Private | Field-encoded full name |
| `dob` | Private | YYYYMMDD integer |
| `nationality` | Private | ISO 3166-1 numeric code |
| `document_id` | Private | Field-encoded document ID |
| `expiry` | Private | YYYYMMDD integer |
| `merkle_path[20]` | Private | Sibling hashes along the Merkle path |
| `merkle_indices[20]` | Private | Left/right bit at each level |
| `merkle_root` | **Public** | Merkle root (from on-chain issuer account) |
| `current_date` | **Public** | Reference date for age/expiry checks |
| `min_age` | **Public** | Minimum age threshold |
| `expected_nationality_hash` | **Public** | `Poseidon(nationality_code)` |

### Output

| Signal | Description |
|---|---|
| `valid` | 1 if all constraints pass |

## Quick Start

### Prerequisites

- **circom 2.1+**: [Install guide](https://docs.circom.io/getting-started/installation/)
- **Node.js 18+**
- **snarkjs**: Installed via npm

### Setup

```bash
cd circuits
npm install

# 1. Compile the circuit
npm run compile

# 2. Generate input (optional — modify scripts/generate_input.js for your data)
node scripts/generate_input.js

# 3. Trusted setup (Powers of Tau + Phase 2)
npm run setup

# 4. Generate witness
npm run witness

# 5. Generate proof
npm run prove

# 6. Verify proof
npm run verify

# Or all at once (after compile + setup):
npm run all
```

## Integration with Flutter App

The Flutter app uses the Groth16 proof generated by these circuits. The workflow:

1. **Off-chain**: Run the circuit scripts to generate `proof.json` and `public.json`.
2. **Mobile**: The Flutter app's `ProofService` constructs the witness and calls the WASM prover (snarkjs) via a platform channel or WebAssembly bridge.
3. **On-chain**: The Solana program's `verifier.rs` performs the Groth16 pairing check using `alt_bn128` syscalls.

## Security Notes

- The `sample_input.json` uses placeholder Merkle values. Use `generate_input.js` with `circomlibjs` for production inputs.
- The trusted setup ceremony should use multiple independent contributors in production.
- The age computation uses integer division which gives a conservative (lower-bound) age estimate.
